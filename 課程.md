## 上午課程：路網圖切版與 PHP 資料 API

## 1. 先讀題目，不要直接寫畫面

正式賽路網圖的重點不是好不好看，而是有沒有把題目指定的資訊正確放出來。

題目要求可以拆成下面幾點：

- 路網圖寬度為 `768px`。
- 每一列顯示的站點數量為 `3`，但後台路線管理也會要求可以設定每列站點數。
- 每個站點要有三個區塊：
  - 上方：目前站點狀態。
  - 中間：站點圖樣。
  - 下方：站點名稱。
- 未發車時顯示「未發車」。
- 已發車時顯示車牌。
- 如果已到站，要顯示「已到站」。
- 如果未到站，要顯示「約 X 分鐘」。

所以每一個站點至少需要這些資料：

```js
{
  id: 1,
  name: '台北車站',
  status: 'not_started',
  plate: '',
  minutes: null
}
```

今天我們先固定使用三種狀態：

| 狀態 | `status` | 畫面顯示 |
| --- | --- | --- |
| 未發車 | `not_started` | 未發車 |
| 行駛中 | `running` | 車牌 + 約 X 分鐘 |
| 已到站 | `arrived` | 車牌 + 已到站 |

## 2. 第一版：先用 HTML/CSS 切出一格站點

先不要急著用 Vue。

先確認一格站點長什麼樣子。

```html
<div class="station">
  <div class="station-status">C12345<br>約 1 分鐘</div>
  <div class="station-dot"></div>
  <div class="station-name">台大醫院</div>
</div>
```

這一格有三個重點：

- `station-status`：放狀態文字。
- `station-dot`：放站點圖樣。
- `station-name`：放站名。

基本 CSS：

```css
.station {
  position: relative;
  min-height: 140px;
  padding: 0 10px;
  text-align: center;
}

.station::before,
.station::after {
  content: "";
  position: absolute;
  top: 0;
  width: 50%;
  height: 8px;
  background: #2f80ed;
}

.station::before {
  left: 0;
}

.station::after {
  right: 0;
}

.station-status {
  position: absolute;
  left: 0;
  right: 0;
  top: -78px;
  min-height: 42px;
  font-size: 14px;
  line-height: 1.5;
}

.station-dot {
  position: absolute;
  left: 50%;
  top: 0;
  transform: translate(-50%, -50%);
  z-index: 2;
  width: 34px;
  height: 34px;
  border-radius: 50%;
  background: #2f80ed;
  border: 4px solid #dbeafe;
}

.station-name {
  position: absolute;
  left: 0;
  right: 0;
  top: 34px;
  font-weight: bold;
}
```

`station-status` 設定 `min-height` 是為了讓每一格高度穩定。

有些站點只顯示「未發車」，有些站點顯示兩行文字，如果不固定高度，畫面會上下跳動。

`station::before` 和 `station::after` 是左右兩段線，線放在站點格子的上緣。

站點圓點用 `position: absolute` 疊在線上。

狀態文字放在線上方，站名放在線下方。

這樣畫面會變成「線與點連接」的路網圖，而不是一格一格的卡片。

## 3. 第二版：用 Grid 排成路網圖

路網圖外層寬度固定為 `768px`。

```html
<div class="network">
  <div class="station">...</div>
  <div class="station">...</div>
  <div class="station">...</div>
</div>
```

```css
.network {
  width: 768px;
  max-width: 100%;
  margin: 24px auto;
  padding-top: 92px;
  display: grid;
  grid-template-columns: repeat(var(--per-row), 1fr);
  border: 2px solid #222;
  --per-row: 3;
  --line-color: #2f80ed;
}

.station.route-start::before,
.station.route-end-left::before,
.station.route-end-right::after {
  display: none;
}

.station.turn-right {
  border-right: 8px solid var(--line-color);
}

.station.turn-left {
  border-left: 8px solid var(--line-color);
}
```

`grid-template-columns: repeat(var(--per-row), 1fr)` 的意思是：

把一列切成 `perRow` 欄，每欄一樣寬。

這會對應正式賽的「每一列顯示的站點數量為 3」。

路網圖不是單純格子。

每個站點用 `::before` 和 `::after` 畫左右線，第一站把左線藏起來，最後一站依照結束方向藏起來。

如果一列結束後要接到下一列，就在轉折站點加上 `turn-right` 或 `turn-left`。

轉折線不要畫在站點中心，否則會穿過站名或狀態文字。

比較好的做法是像參考路網圖一樣，把轉折線放在整格的右邊框或左邊框。

這樣仍然使用 Grid 排版，但視覺上會是「線與點連接」的路網。

## 4. 第三版：用 Vue 資料產生站點

如果每一站都手刻 HTML，後台資料改了，前台不可能跟著更新。

正式賽需要的是資料驅動畫面。

先準備站點資料：

```js
const stations = ref([
  {
    id: 1,
    name: '台北車站',
    status: 'not_started',
    plate: '',
    minutes: null
  },
  {
    id: 2,
    name: '西門',
    status: 'running',
    plate: 'C12345',
    minutes: 1
  },
  {
    id: 3,
    name: '龍山寺',
    status: 'arrived',
    plate: 'C12345',
    minutes: 0
  }
])
```

HTML 改成 `v-for`：

```html
<!-- v-for：把 stations 陣列中的每一筆站點資料轉成一格 station 畫面。 -->
<!-- :key：用 station.id 當每一格的身份，讓 Vue 更新列表時能穩定追蹤。 -->
<div class="station" v-for="station in stations" :key="station.id">
  <div class="station-status">
    <template v-if="station.status === 'not_started'">
      未發車
    </template>

    <template v-else-if="station.status === 'running'">
      {{ station.plate }}<br>
      約 {{ station.minutes }} 分鐘
    </template>

    <template v-else-if="station.status === 'arrived'">
      {{ station.plate }}<br>
      已到站
    </template>
  </div>

  <div class="station-dot"></div>
  <div class="station-name">{{ station.name }}</div>
</div>
```

這裡的判斷不要寫得太花。

比賽時最重要的是資料狀態清楚，畫面可以穩定顯示。

## 5. 用資料控制每列站點數

正式賽題目目前示意圖是每列 3 個站點。

但路線管理功能又要求每條路線可以設定「前台每列顯示數量」。

所以我們不要把 `3` 寫死在 CSS。

Vue 資料：

```js
const perRow = ref(3)
```

HTML：

```html
<div class="network" :style="{ '--per-row': perRow }">
  ...
</div>
```

這段的意思是：

當 `perRow` 是 `3`，CSS 變數就會讓 Grid 變成：

```css
grid-template-columns: repeat(3, 1fr);
```

當 `perRow` 是 `4`，Grid 就會變成：

```css
grid-template-columns: repeat(4, 1fr);
```

這就是正式賽很常見的做法：

資料庫或 API 給設定值，前端畫面根據設定值改變。

如果要做成蛇行路線，可以先把站點依照 `perRow` 切成多列，再把第二列、第四列反轉顯示。

這樣資料順序仍然是正常路線順序，但畫面會像題目示意圖一樣折返。

有一個很重要的細節：

如果反向列是最後一列，而且站點數不足一整列，要先補空白格，讓站點靠右排列。

例如每列 3 站，但總共只有 5 站：

```text
第 1 列：台北車站  西門    龍山寺
第 2 列：空白      新埔    江子翠
```

這樣右邊轉折線才會接到第二列的右側站點。

如果不補空白格，第二列會從左邊開始排，右邊轉折線就會落空。

## 6. PHP 回傳路網圖 JSON

接著把資料從前端搬到 PHP。

建立 `api/network.php`。

這支 API 的目標：

- 回傳固定路線名稱。
- 回傳 `perRow`。
- 回傳站點資料。
- 每次重新整理或重新 fetch 時，用亂數讓站點數量和狀態有一點變化。

範例：

```php
<?php
header('Content-Type: application/json; charset=utf-8');

$allStations = [
    '台北車站',
    '西門',
    '龍山寺',
    '江子翠',
    '新埔',
    '板橋',
    '府中',
    '亞東醫院',
    '海山',
    '土城',
    '永寧'
];

$count = rand(5, count($allStations));
$names = array_slice($allStations, 0, $count);
$busIndex = rand(0, $count - 1);
$plate = 'C' . rand(10000, 99999);
$perRow = rand(2, 4);

$stations = [];

foreach ($names as $index => $name) {
    if ($index < $busIndex) {
        $status = 'arrived';
        $minutes = 0;
    } elseif ($index === $busIndex) {
        $status = 'running';
        $minutes = rand(1, 5);
    } else {
        $status = 'not_started';
        $minutes = null;
    }

    $stations[] = [
        'id' => $index + 1,
        'name' => $name,
        'status' => $status,
        'plate' => $status === 'not_started' ? '' : $plate,
        'minutes' => $minutes
    ];
}

echo json_encode([
    'routeName' => '板南線',
    'perRow' => $perRow,
    'stations' => $stations
], JSON_UNESCAPED_UNICODE | JSON_PRETTY_PRINT);
```

這支 API 不是正式版資料庫 API。

它是上午用來練習「前端向 PHP 拿資料，畫面跟著資料變化」的練習 API。

## 7. Vue fetch PHP 路網圖資料

建立 `03-network-fetch.html`。

核心資料：

```js
const routeName = ref('')
const perRow = ref(3)
const stations = ref([])
const isLoading = ref(false)
```

載入資料函式：

```js
/*
函式英文名稱：loadNetwork
中文直譯名稱：載入路網
帶入項：無
回傳值：無
具體用途：向 api/network.php 取得路線名稱、每列站點數與站點狀態，並更新畫面。
會改動的資料：routeName、perRow、stations、isLoading
*/
function loadNetwork() {
  isLoading.value = true

  fetch('./api/network.php')
    // 第一個 then 拿到的是回應物件，還不是可直接使用的路網資料。
    .then(function(response) {
      return response.json()
    })
    // 第二個 then 才拿到 PHP 回傳的 JSON 內容。
    .then(function(data) {
      routeName.value = data.routeName
      perRow.value = data.perRow
      stations.value = data.stations
    })
    .catch(function(error) {
      console.log(error)
    })
    .finally(function() {
      isLoading.value = false
    })
}
```

按鈕：

```html
<!-- @click：按下後重新向 PHP 拿一次資料，因為 PHP 有亂數，所以畫面會變化。 -->
<button @click="loadNetwork">
  重新取得路網資料
</button>
```

啟動時先載入一次：

```js
loadNetwork()
```

這裡請學生觀察：

- 按一次按鈕，車牌可能會換。
- 按一次按鈕，公車位置可能會換。
- 按一次按鈕，站點數量可能會換。
- 前端沒有自己亂數，變化來自 PHP 回傳資料。

## 8. 為什麼下午要做 CRUD

正式賽後台管理包含：

- 路線管理

這三個管理功能的本質都是 CRUD。

CRUD 是四個動作：

| 動作 | 英文 | 常見功能 |
| --- | --- | --- |
| 新增 | Create | 新增站點、新增車輛、新增路線 |
| 讀取 | Read | 列出所有資料 |
| 修改 | Update | 修改站名、修改已行駛時間 |
| 刪除 | Delete | 刪除一筆資料 |

下午會從資料最簡單的站點開始，再做車輛，最後做路線。

## 9. 資料庫結構

正式賽的資料不是一張表就能放完。

站點、路線、車輛彼此有關係。

今天使用四張表：

```sql
CREATE DATABASE IF NOT EXISTS ptqs
  DEFAULT CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE ptqs;

CREATE TABLE stations (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50) NOT NULL
);

CREATE TABLE routes (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(50) NOT NULL,
  per_row INT NOT NULL DEFAULT 3
);

CREATE TABLE route_stations (
  id INT AUTO_INCREMENT PRIMARY KEY,
  route_id INT NOT NULL,
  station_id INT NOT NULL,
  sort_order INT NOT NULL,
  drive_time INT NOT NULL DEFAULT 1,
  stop_time INT NOT NULL DEFAULT 1
);

CREATE TABLE buses (
  id INT AUTO_INCREMENT PRIMARY KEY,
  route_id INT NOT NULL,
  plate VARCHAR(20) NOT NULL,
  runtime INT NOT NULL DEFAULT 0
);
```

### 每張表的用途

`stations` 放站點。

```text
1 台北車站
2 台大醫院
3 大安森林公園
```

`routes` 放路線。

```text
1 北車 - 101 每列 3 站
```

`route_stations` 放「某條路線有哪些站、順序是什麼、每段時間是多少」。

```text
北車 - 101 第 1 站 台北車站 行駛 2 分鐘 停留 1 分鐘
北車 - 101 第 2 站 台大醫院 行駛 3 分鐘 停留 1 分鐘
```

`buses` 放車輛。

```text
C12345 在 北車 - 101 路線，已行駛 5 分鐘
```

`route_stations` 是中介表。

它存在的原因是：

一個站點可以出現在多條路線中，一條路線也會有多個站點。

## 11. PDO 連線

建立 `api/db.php`。

```php
<?php
$host = 'localhost';
$dbname = 'ptqs';
$user = 'root';
$pass = '';

$pdo = new PDO(
    "mysql:host=$host;dbname=$dbname;charset=utf8mb4",
    $user,
    $pass,
    [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC
    ]
);
```

這支檔案只做一件事：

建立 `$pdo`，給其他 API 檔案使用。

`PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION` 的意思是：

SQL 出錯時，讓 PHP 用明確錯誤告訴我們，而不是安靜失敗。

## 12. API 統一讀取 JSON 輸入

新增與修改資料時，前端會送 JSON 到 PHP。

PHP 可以這樣讀：

```php
$input = json_decode(file_get_contents('php://input'), true);
```

這一行的意思是：

- `php://input`：讀取前端送來的原始內容。
- `json_decode(..., true)`：把 JSON 轉成 PHP 陣列。

今天 API 回傳也都用 JSON：

```php
header('Content-Type: application/json; charset=utf-8');
```

## 13. 站點 CRUD API

建立 `api/stations.php`。

站點資料最單純，只需要 `id` 和 `name`。

### 讀取所有站點

```php
if ($_SERVER['REQUEST_METHOD'] === 'GET') {
    $stmt = $pdo->query('SELECT * FROM stations ORDER BY id ASC');
    echo json_encode([
        'stations' => $stmt->fetchAll()
    ], JSON_UNESCAPED_UNICODE);
}
```

### 新增站點

```php
if ($_SERVER['REQUEST_METHOD'] === 'POST') {
    $input = json_decode(file_get_contents('php://input'), true);

    $stmt = $pdo->prepare('INSERT INTO stations (name) VALUES (?)');
    $stmt->execute([$input['name']]);

    echo json_encode([
        'success' => true
    ], JSON_UNESCAPED_UNICODE);
}
```

### 修改站點

```php
if ($_SERVER['REQUEST_METHOD'] === 'PUT') {
    $input = json_decode(file_get_contents('php://input'), true);

    $stmt = $pdo->prepare('UPDATE stations SET name = ? WHERE id = ?');
    $stmt->execute([$input['name'], $input['id']]);

    echo json_encode([
        'success' => true
    ], JSON_UNESCAPED_UNICODE);
}
```

### 刪除站點

```php
if ($_SERVER['REQUEST_METHOD'] === 'DELETE') {
    $input = json_decode(file_get_contents('php://input'), true);

    $stmt = $pdo->prepare('DELETE FROM stations WHERE id = ?');
    $stmt->execute([$input['id']]);

    echo json_encode([
        'success' => true
    ], JSON_UNESCAPED_UNICODE);
}
```

正式做檔案時，可以把這四段放在同一支 `stations.php`，用 `$_SERVER['REQUEST_METHOD']` 判斷目前是哪一種動作。

## 14. 站點管理前端

站點管理頁需要：

- 站點列表
- 新增輸入框
- 編輯狀態
- 刪除按鈕

Vue 資料：

```js
const stations = ref([])
const name = ref('')
const editId = ref(null)
```

載入站點：

```js
/*
函式英文名稱：loadStations
中文直譯名稱：載入站點
帶入項：無
回傳值：無
具體用途：向 stations.php 取得所有站點，放進 stations 陣列讓 v-for 顯示。
會改動的資料：stations
*/
function loadStations() {
  fetch('./api/stations.php')
    .then(function(response) {
      return response.json()
    })
    .then(function(data) {
      stations.value = data.stations
    })
}
```

新增站點：

```js
/*
函式英文名稱：addStation
中文直譯名稱：新增站點
帶入項：無
回傳值：無
具體用途：把 name 送到 stations.php，新增成功後重新載入站點列表。
會改動的資料：name、stations
*/
function addStation() {
  fetch('./api/stations.php', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: name.value
    })
  })
    .then(function(response) {
      return response.json()
    })
    .then(function() {
      name.value = ''
      loadStations()
    })
}
```

修改和刪除的寫法和新增很像。

差別只在：

- 修改使用 `PUT`。
- 刪除使用 `DELETE`。
- 修改和刪除都要送出 `id`。

## 15. 車輛 CRUD

車輛比站點多一個重點：

車輛要屬於某一條路線。

所以 `buses` 表會有 `route_id`。

車輛管理需要的欄位：

- 路線
- 車牌
- 已行駛時間

### 車輛列表 SQL

列表頁通常不只想看到 `route_id`，也想看到路線名稱。

所以會用 `JOIN`。

```sql
SELECT
  buses.id,
  buses.plate,
  buses.runtime,
  routes.name AS route_name
FROM buses
JOIN routes ON buses.route_id = routes.id
ORDER BY buses.id ASC
```

`JOIN` 的意思是：

把車輛資料和路線資料接在一起。

這樣畫面就能顯示：

```text
北車 - 101 / C12345 / 已行駛 5 分鐘
```

而不是只看到：

```text
route_id = 1
```

### 新增車輛

前端需要送：

```js
{
  route_id: 1,
  plate: 'C12345',
  runtime: 5
}
```

PHP：

```php
$stmt = $pdo->prepare(
    'INSERT INTO buses (route_id, plate, runtime) VALUES (?, ?, ?)'
);
$stmt->execute([
    $input['route_id'],
    $input['plate'],
    $input['runtime']
]);
```

### 修改車輛

正式賽題目要求車輛編輯時主要修改「已行駛時間」。

所以修改 API 可以先只更新 `runtime`。

```php
$stmt = $pdo->prepare('UPDATE buses SET runtime = ? WHERE id = ?');
$stmt->execute([
    $input['runtime'],
    $input['id']
]);
```

這樣比較貼近題目，也不會讓下午課程分散。

## 16. 路線 CRUD 的資料重點

路線 CRUD 是下午最重要也最容易卡住的地方。

因為路線不是只有一個名稱。

路線包含：

- 路線名稱
- 前台每列顯示站點數
- 多個站點
- 每個站點的順序
- 每段行駛時間
- 每站停留時間

所以路線會用兩張表一起處理：

- `routes`
- `route_stations`

新增一條路線時，流程是：

1. 新增 `routes`。
2. 取得剛新增的 `route_id`。
3. 把這條路線的站點逐筆新增到 `route_stations`。

---

## 17. 路線新增資料格式

前端送給 PHP 的資料可以長這樣：

```js
{
  name: '北車 - 101',
  per_row: 3,
  stations: [
    {
      station_id: 1,
      sort_order: 1,
      drive_time: 2,
      stop_time: 1
    },
    {
      station_id: 2,
      sort_order: 2,
      drive_time: 3,
      stop_time: 1
    }
  ]
}
```

注意這裡的 `stations` 不是站點資料表本身。

它代表「這條路線選了哪些站，以及每個站在路線中的設定」。

---

## 18. 新增路線 PHP 流程

```php
$input = json_decode(file_get_contents('php://input'), true);

$stmt = $pdo->prepare('INSERT INTO routes (name, per_row) VALUES (?, ?)');
$stmt->execute([
    $input['name'],
    $input['per_row']
]);

$routeId = $pdo->lastInsertId();

$stmt = $pdo->prepare(
    'INSERT INTO route_stations
    (route_id, station_id, sort_order, drive_time, stop_time)
    VALUES (?, ?, ?, ?, ?)'
);

foreach ($input['stations'] as $station) {
    $stmt->execute([
        $routeId,
        $station['station_id'],
        $station['sort_order'],
        $station['drive_time'],
        $station['stop_time']
    ]);
}

echo json_encode([
    'success' => true
], JSON_UNESCAPED_UNICODE);
```

`lastInsertId()` 很重要。

因為你新增 `routes` 之後，資料庫才會產生這條路線的 `id`。

接著才能把這個 `id` 放進 `route_stations.route_id`。

---

## 19. 修改路線 PHP 流程

修改路線時，最簡單穩定的做法是：

1. 更新 `routes` 的名稱與 `per_row`。
2. 刪除這條路線原本的 `route_stations`。
3. 重新新增新的 `route_stations`。

```php
$stmt = $pdo->prepare('UPDATE routes SET name = ?, per_row = ? WHERE id = ?');
$stmt->execute([
    $input['name'],
    $input['per_row'],
    $input['id']
]);

$stmt = $pdo->prepare('DELETE FROM route_stations WHERE route_id = ?');
$stmt->execute([$input['id']]);

$stmt = $pdo->prepare(
    'INSERT INTO route_stations
    (route_id, station_id, sort_order, drive_time, stop_time)
    VALUES (?, ?, ?, ?, ?)'
);

foreach ($input['stations'] as $station) {
    $stmt->execute([
        $input['id'],
        $station['station_id'],
        $station['sort_order'],
        $station['drive_time'],
        $station['stop_time']
    ]);
}
```

這個做法不是資料庫理論上最精細的做法。

但它很適合比賽：

- 程式比較直。
- 不容易寫錯。
- 路線站點數通常不大。
- 畫面修改後可以整包重存。

## 20. 讀取單一路線詳細資料

編輯路線時，需要拿到路線本身與路線中的站點。

SQL：

```sql
SELECT
  routes.id AS route_id,
  routes.name AS route_name,
  routes.per_row,
  route_stations.station_id,
  stations.name AS station_name,
  route_stations.sort_order,
  route_stations.drive_time,
  route_stations.stop_time
FROM routes
JOIN route_stations ON routes.id = route_stations.route_id
JOIN stations ON route_stations.station_id = stations.id
WHERE routes.id = ?
ORDER BY route_stations.sort_order ASC
```

這段會把一條路線的每一站都查出來。

前端拿到後，就可以放回編輯表單。

## 21. 路線管理前端資料

路線表單至少需要：

```js
const routeName = ref('')
const perRow = ref(3)
const allStations = ref([])
const routeStations = ref([])
```

`allStations` 是所有可選站點。

`routeStations` 是目前這條路線已經選進來的站點。

加入路線站點時，可以先放一筆預設資料：

```js
function addRouteStation() {
  routeStations.value.push({
    station_id: '',
    sort_order: routeStations.value.length + 1,
    drive_time: 1,
    stop_time: 1
  })
}
```

送出時：

```js
function saveRoute() {
  fetch('./api/routes.php', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json'
    },
    body: JSON.stringify({
      name: routeName.value,
      per_row: perRow.value,
      stations: routeStations.value
    })
  })
    .then(function(response) {
      return response.json()
    })
    .then(function() {
      routeName.value = ''
      perRow.value = 3
      routeStations.value = []
      loadRoutes()
    })
}
```

比賽時路線管理最重要的是資料送得正確。

畫面可以先簡單，但每一站的 `station_id`、`sort_order`、`drive_time`、`stop_time` 要完整。

## 常見錯誤

這裡只列今天最容易影響完成度的錯誤。

### 1. PHP 沒有設定 JSON header

```php
header('Content-Type: application/json; charset=utf-8');
```

少了這行，有時前端仍然能讀，但除錯會比較混亂。

### 2. `fetch` 忘記 `return response.json()`

錯誤寫法：

```js
.then(function(response) {
  response.json()
})
```

正確寫法：

```js
.then(function(response) {
  return response.json()
})
```

### 3. PDO SQL 欄位名稱和資料表不一致

例如資料表欄位叫 `per_row`，PHP 卻寫成 `perRow`。

資料庫欄位和前端變數可以不同，但轉換時要看清楚。

### 4. 路線新增後忘記 `lastInsertId()`

如果沒有取得新路線的 `id`，就不知道 `route_stations.route_id` 要填什麼。

### 5. 路線站點順序沒有存

只存站點是不夠的。

路線一定要知道第一站、第二站、第三站。

所以 `sort_order` 一定要存。
